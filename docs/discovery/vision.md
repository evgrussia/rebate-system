# Vision Document: CryptoRebate

> Партнёрский rebate-сервис для криптовалютных бирж

---

## 1. Problem Statement (Проблема)

### Для трейдеров

Криптотрейдеры ежедневно платят существенные комиссии биржам за спотовые и фьючерсные сделки. При активной торговле комиссии могут составлять сотни и тысячи долларов в месяц. При этом:

- **Трейдеры не знают** о существовании rebate/cashback-программ у бирж, либо не используют их из-за сложности.
- **Нет единой точки входа** — чтобы получать rebate на нескольких биржах, нужно отдельно разбираться с партнёрской программой каждой.
- **Нет прозрачности** — трейдеры не видят, сколько именно комиссий они платят и сколько могли бы возвращать.

### Для рынка

- Партнёрские программы бирж существуют, но ориентированы на инфлюенсеров и affiliate-маркетологов, а не на конечных трейдеров.
- Нет удобного агрегатора, который объединял бы rebate-программы нескольких бирж в одном интерфейсе.

### Суть проблемы

> Криптотрейдеры переплачивают комиссии, потому что не имеют простого инструмента для получения rebate сразу на всех используемых биржах.

## 2. Value Proposition (Ценностное предложение)

### Для трейдеров

**«Зарегистрируйся один раз — получай cashback с комиссий на всех биржах»**

- Единая регистрация через сервис даёт доступ к rebate-программам 11+ бирж
- Прозрачная статистика: сколько наторговал, сколько комиссий заплатил, сколько вернули
- Простой вывод заработанного в криптовалюте
- Полноценный личный кабинет: веб-интерфейс + Telegram-бот

### Для бизнеса

- Масштабируемый источник дохода: маржа = (rebate от биржи) − (выплата трейдеру)
- Сетевой эффект: больше трейдеров → больший объём торгов → больше комиссий → больше дохода
- Низкая стоимость привлечения: трейдеры мотивированы cashback'ом
- Расширяемость: добавление новых бирж увеличивает привлекательность без значительных затрат

## 4. Success Metrics (Метрики успеха)

### Ключевые KPIs для MVP

| Метрика | Описание | Целевое значение (MVP) |
|---------|----------|------------------------|
| **Registered Users** | Количество зарегистрированных трейдеров | 1,000+ |
| **Active Traders** | Трейдеры с объёмом торгов > $0 за последние 30 дней | 30% от зарегистрированных |
| **Total Trading Volume** | Совокупный объём торгов привлечённых трейдеров | $10M+/мес |
| **Gross Rebate Revenue** | Общая сумма rebate, полученная от бирж | Зависит от условий бирж |
| **Net Revenue (Margin)** | Маржа сервиса (rebate − выплаты) | >0 (unit economics положительна) |
| **Payout Rate** | Доля rebate, выплаченная трейдерам | 50–80% (настраиваемо) |
| **Exchanges Connected** | Количество подключённых бирж | 11 |
| **Avg. Rebate per User** | Средний rebate на одного трейдера в месяц | $10+ |
| **User Retention (M1)** | Удержание после первого месяца | >60% |
| **Withdrawal Success Rate** | Успешность выводов | >99% |

### North Star Metric

> **Total Trading Volume** — совокупный объём торгов привлечённых трейдеров на всех биржах. Эта метрика напрямую коррелирует с доходом сервиса.

### Ключевые компоненты

| Компонент | Описание |
|-----------|----------|
| **Веб-ЛК трейдера** | React-приложение: регистрация, дашборд, статистика, вывод средств |
| **Telegram-бот ЛК** | Полноценный личный кабинет в Telegram: статистика, вывод, уведомления |
| **Admin Panel** | Управление биржами, ставками rebate, пользователями, выплатами |
| **Django Backend** | REST API, бизнес-логика, аутентификация, расчёт rebate |
| **Exchange Integration Layer** | Унифицированный адаптер для API каждой биржи (получение данных о комиссиях) |
| **Rebate Calculator** | Расчёт rebate для каждого трейдера на основе данных бирж и настроек |
| **Payout Engine** | Управление выплатами: очередь, проверка, исполнение |
| **PostgreSQL** | Хранение: пользователи, транзакции, выплаты, настройки бирж |

### Пользовательский путь (Happy Path)

```
1. Трейдер находит CryptoRebate (сайт / Telegram / инфлюенсер)
2. Регистрируется в сервисе
3. Получает персональные реферальные ссылки для каждой биржи
4. Регистрируется на биржах по этим ссылкам (или привязывает существующий аккаунт)
5. Торгует как обычно
6. Сервис автоматически получает данные о комиссиях от бирж
7. Rebate рассчитывается и отображается в ЛК
8. Трейдер выводит накопленный rebate в крипте
```

## 6. Competitive Advantage (Конкурентные преимущества)

### Прямые конкуренты

| Конкурент | Модель | Наши преимущества |
|-----------|--------|-------------------|
| Отдельные affiliate-ссылки инфлюенсеров | Одна биржа, нет ЛК, нет прозрачности | Агрегация бирж + полноценный ЛК + прозрачность |
| Сервисы типа CoinTracker/3Commas | Трекинг/боты, не rebate | Фокус именно на cashback с комиссий |
| Прямые партнёрские программы бирж | Для аффилиатов, не для трейдеров | Удобный интерфейс для конечного трейдера |

### Ключевые конкурентные преимущества

1. **Агрегация 11+ бирж** — единый ЛК для всех rebate, не нужно управлять каждой биржей отдельно
2. **Прозрачность** — трейдер видит точные данные: объём торгов, комиссии, начисленный и выплаченный rebate
3. **Два канала ЛК** — веб + Telegram-бот для максимального удобства
4. **Простота** — не нужно разбираться в affiliate-программах, достаточно зарегистрироваться и торговать
5. **Глобальный охват** — сервис работает по всему миру, поддержка множества бирж
6. **Автоматизация** — rebate рассчитывается и начисляется автоматически, без ручного вмешательства

### Барьеры для входа (Moat)

- Интеграция с 11 биржами — значительный технический барьер для копирования
- Накопленная база трейдеров и торговых объёмов
- Сетевой эффект: чем больше пользователей, тем лучше условия от бирж

## 7. Risks & Assumptions (Риски и допущения)

### Ключевые риски

| # | Риск | Вероятность | Влияние | Митигация |
|---|------|-------------|---------|-----------|
| R1 | Биржа изменит условия партнёрской программы | Высокая | Высокое | Диверсификация по 11 биржам, мониторинг условий |
| R2 | API биржи нестабилен / меняется | Средняя | Среднее | Абстрактный адаптер, мониторинг, fallback |
| R3 | Регуляторные ограничения в отдельных странах | Средняя | Среднее | Гибкая гео-политика, юридический мониторинг |
| R4 | Низкий трафик / медленное привлечение | Средняя | Высокое | Партнёрство с инфлюенсерами, SEO, контент-маркетинг |
| R5 | Конкуренты скопируют модель | Низкая | Среднее | Скорость выхода, UX, качество интеграций |
| R6 | Мошеннические аккаунты / wash trading | Средняя | Среднее | Антифрод-система, верификация через биржу |

### Допущения

1. Все 11 бирж имеют открытые или доступные API для партнёрских программ
2. Условия бирж позволяют выплачивать часть rebate конечным трейдерам
3. Трейдеры готовы регистрироваться на биржах по реферальным ссылкам сервиса
4. Сервис сможет получать данные о комиссиях трейдеров через API бирж

## 8. Scope границы (MVP vs Future)

### MVP (v1.0)

- 11 бирж подключены
- Веб-ЛК трейдера (React)
- Telegram-бот ЛК
- Admin Panel
- Автоматический расчёт rebate
- Вывод средств в крипте
- Лендинг / маркетинговый сайт

### V2.0 (Future)

- Мобильное приложение (React Native)
- Расширенная аналитика для трейдеров (графики доходности, сравнение бирж)
- Автоматический арбитраж rebate-ставок между биржами
- Дополнительные биржи
- VIP-программа для крупных трейдеров
- API для партнёров / виджеты для сайтов

### Не планируется

- Суб-партнёрская / multi-level программа
- Собственная торговая платформа
- Copy-trading
- Фиатные выводы
- KYC на стороне сервиса
